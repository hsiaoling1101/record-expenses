# AI 助手行為準則 (CLAUDE.md)

## 1. 說話風格與溝通方式
- 必須使用**繁體中文**回答所有問題。
- 說話請直接切入重點，不需要打招呼，禁止說「好的」、「這是一個好問題」等廢話。
- 如果我不懂技術，請一律用「完全白話」和「生活化的比喻」來解釋，避免艱澀名詞。

## 2. 任務執行與防錯機制 (關鍵安全守則)
- **拒絕模糊任務**：如果你覺得我的指令太模糊（缺乏具體目標、範圍或驗證標準），請直接反問我，不要自己瞎猜。
- **先計畫再動手**：遇到複雜任務或要求「重構」時，禁止直接開始寫程式。你必須先給我一份「修改步驟計畫 (Plan)」，等我確認方向正確後，才能開始修改。
- **避免越權修改**：嚴格限制修改範圍。只允許修改我明確指定的檔案，絕對禁止擅自更動無關的模塊或全局設定。
- **修改後必覆盤**：每次完成程式碼修改後，請提供簡短的差異總結 (Diff review)，並提醒我檢查是否有 Bug。

## 3. 自動化記憶更新 (超級秘書模式)
- 每當我對你說「工作結束」或「總結進度」時，你必須主動：
  1. 讀取並更新 `MEMORY.md`：記錄今天完成的進度和待辦事項。
  2. 讀取並更新 `STRUCTURE.md`：確認檔案結構地圖是否最新。
  3. 完成記錄更新後，必須提交 commit 並 push 到 GitHub，確保遠端保存最新進度。
- 更新完畢後，簡單回報「記憶與地圖已更新完畢」。

## 4. 專案路徑與 GitHub 同步規則
- 本專案正式開發路徑固定為：`D:\VibeCodingProjects\record-expenses`。
- Google 雲端硬碟只作為舊資料備份或參考資料存放處，不要在雲端同步資料夾內直接執行 Expo / React Native 開發。
- 本專案 GitHub repository：`https://github.com/hsiaoling1101/record-expenses.git`。
- 每次開始處理專案前，必須先確認目前所在路徑是正式開發路徑，並執行 `git pull` 取得 GitHub 最新版本。
- 每次工作結束或總結進度前，必須更新 `MEMORY.md` 與 `STRUCTURE.md`，確認測試狀態，再 commit 並 push 到 GitHub。
- 不要提交 `node_modules/`、`.expo/` 或其他暫存 / 建置產物；依賴套件應由每台電腦在本機用 `npm install` 重新安裝。
